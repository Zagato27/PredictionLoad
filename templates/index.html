Входной JSON со ступенями
Целевой поток запросов (RPS > 0). Используется для прогнозов и рекомендаций.
SLO по максимальной задержке (> 0 мс). Строится горизонтальная линия на графике задержек.
Подсказки

Вы можете выбрать готовый набор данных в выпадающем списке над редактором JSON. Редактор подсвечивает синтаксис и проверяет валидность JSON; кнопка «Рассчитать прогноз» блокируется при ошибке. Кнопка «Форматировать JSON» приведёт документ к читаемому виду.

Ступени (steps[])
  • step: string|int — идентификатор шага (например, "s1" или 1).
  • rps: number > 0 — запросов в секунду на шаге.
  • avg_ms, max_ms: ms — средняя и максимальная задержки; max_ms ≥ avg_ms.
  • pods: integer ≥ 1 — число подов сервиса в этом шаге.
  • cpu_usage_m: millicores — суммарный фактический CPU usage сервиса на шаге.
  • cpu_request_m_per_pod, cpu_limit_m_per_pod: millicores на под.
  • mem_workingset_mib: MiB — фактическая память (workingset) суммарно.
  • mem_request_mib_per_pod, mem_limit_mib_per_pod: MiB на под.
  • net_in_mbps, net_out_mbps: MB/s — входящий/исходящий трафик.
  • concurrency_optional: integer ≥ 0 — одновременность (если известна).
  • Обратная совместимость: поля-доли cpu_util, ram_util могут использоваться, если абсолютные K8s поля не заданы.
Цель (target)
  • target_rps: number > 0 — целевой RPS для прогноза.
  • slo_ms_max_optional: number > 0 — SLO по максимальной задержке (мс).
Емкость/пороги (capacity)
  • u_max_cpu, u_max_ram: (0..1] — целевые утилизации относительно requests (HPA-логика).
  • net_capacity_in_mbps_optional, net_capacity_out_mbps_optional: > 0 — доступная полоса по сети (MB/s).
  • mmc_c_optional: integer ≥ 1 — число серверов в модели M/M/c (для очередей/ожидания).
Моделирование (modeling)
  • use_m_m_1: включить M/M/1 (1 сервер). Быстрый базовый ориентир для однопоточных сервисов. Показательна при низкой нагрузке; нестабильна при ρ ≥ 1.
  • use_m_m_c: включить M/M/c (несколько серверов). Учитывает ожидание в очереди; подходит для сервисов с несколькими воркерами/подами. Использует capacity.mmc_c_optional как число серверов c.
  • use_kingman: включить G/G/1 (Kingman). Учитывает вариативность входа и обслуживания; улучшает оценку задержек для одного сервера при «неровной» нагрузке (требует ρ < 1).
  • use_g_g_c: включить G/G/c (Allen–Cunneen). Практичная аппроксимация для нескольких серверов/подов с вариативностью. Также использует capacity.mmc_c_optional как c.
  • Можно включать несколько моделей одновременно — результаты будут показаны вместе в таблице и на графиках.
